# TypeScript学習ロードマップ

このファイルは、TypeScript学習の推奨順序と各Dayで学ぶテーマを定義します。
問題作成スキルはこのロードマップを参照して、次に学ぶべきテーマを自動的に提案します。

## 基礎編（Day01-20）

TypeScriptの基本的な型システムをじっくり固めます。初心者はこの期間で焦らず確実に理解を深めてください。

## ロードマップのカスタマイズ

このロードマップはあくまで推奨順序です。以下のような調整が可能です：

### テーマのスキップ

特定のテーマをスキップしたい場合は、問題作成時にユーザーが別のテーマを指定してください。

### テーマの追加

新しいテーマを学習したい場合は、このファイルに追加してください：

```markdown
| XX | 新しいテーマ | `functionName` | 学習内容の説明 |
```

### 難易度の調整

- 基礎編が簡単すぎる場合 → Day21から始める
- 基礎編が難しい場合 → Day01-20をゆっくり繰り返す
- 中級編に進む前に → Day01-20の総復習を推奨
- 上級編に進む前に → Day21-40の総復習を推奨

## 学習の進め方

### 推奨ペース

- **初心者（毎日1問）**: 60日で完走
- **週3-4問**: 4-5ヶ月で完走（無理のないペース）
- **集中学習（週5問）**: 3ヶ月で完走

### マイルストーン

- **Day20完了**: 基礎編修了 → 簡単なCLIツールを作成
- **Day40完了**: 中級編修了 → REST APIクライアントを実装
- **Day60完了**: 上級編修了 → 独自の型ライブラリを作成

### 復習のタイミング

- Day10完了後 → Day01-10を振り返る
- Day20完了後 → Day01-20を総復習
- Day30完了後 → Day21-30を振り返る
- Day40完了後 → Day21-40を総復習
- Day50完了後 → Day41-50を振り返る
- Day60完了後 → 全体を総復習

### 実践への応用

各レベル完了後の実践課題：

**基礎編完了後（Day20）**:

- 簡単なCLIツールを作成（引数のパース、ファイル操作）
- REST APIのクライアント実装（型安全なHTTPリクエスト）
- CSVパーサーの実装（型安全なデータ変換）

**中級編完了後（Day40）**:

- Expressでのバックエンド実装（型安全なルーティング）
- React/Vueでのフロントエンド（Props、State の型付け）
- データベースアクセス層の実装（型安全なクエリ）

**上級編完了後（Day60）**:

- 独自のユーティリティ型ライブラリ作成
- 型レベルでのバリデーションライブラリ実装
- OSSへのコントリビューション

## トラブルシューティング

### 問題が難しすぎる場合

1. 前のDayに戻って復習
2. README.mdの「背景知識」セクションを熟読
3. ヒントを参考にしながら段階的に実装
4. 公式ドキュメントや参考書で補完学習

### 問題が簡単すぎる場合

1. より複雑なケースを自分で追加
2. エッジケースのテストを追加
3. パフォーマンス最適化に挑戦
4. 次のフェーズに進む

### モチベーション維持

- GitHubにコミットして進捗を可視化
- 学習記録をつける（何を学んだか、つまづいた点）
- SNSで学習内容をシェア
- 定期的に実践課題に取り組む
